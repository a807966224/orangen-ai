# 按局数完成目标系统输赢金额算法 - 需求规格说明书 (SRS)

## 1. 文档目的

本文档定义**按局数完成目标系统输赢金额算法**的功能需求、验收标准及测试用例。
该算法用于控制游戏系统中奖率分布，确保系统在上指定局数内达到目标输赢金额。

---

## 2. 核心定义

### 2.1 关键术语

| 术语 | 定义 |
|------|------|
| **系统输赢金额** | `总投注 - 总派奖` |
| **目标金额 (targetSystemProfit)** | 可正可负：正=系统赢，负=系统输（玩家赢） |
| **局数 (targetGames)** | 完成目标的局数上限 |
| **中奖率** | 中奖次数 / 总局数 |
| **大奖 (Big Win)** | 倍率 > 20 的中奖 |
| **小奖 (Small Win)** | 0 < 倍率 ≤ 20 的中奖 |

### 2.2 公式

```
系统输赢金额 = Σ(每局投注) - Σ(每局投注 × 对应倍率)
```

## 3. 功能需求

### 3.1 必选需求 (MUST)

| ID | 需求描述 | 验收标准 |
|----|----------|----------|
| **F-01** | 构造函数参数保持不变 | `targetGames`, `targetSystemProfit`, `rateWeightMap`, `vibeType`, `betList`, `gamesPlayed`, `winCount`, `totalBet`, `totalPayout` 参数签名不变 |
| **F-02** | 缪奥计算倍率接口不变 | `calculateMultiplier(playerBet)` 返回 `Double`，入参只有 `playerBet` |
| **F-03** | 总中奖率 ≤ 40% | 实际中奖次数 / 总局数 ≤ 0.40 |
| **F-04** | 大奖占比 = 中奖次数的 50% | 大奖次数 / 中奖次数 = 0.50 (即大奖概率 = 20%，小奖概率 = 20%) |
| **F-05** | 局数内完成目标 | 在 targetGames 局内达到目标金额 |
| **F-06** | 目标偏差 ≤ 10% | `|实际金额 - 目标金额| / |目标金额| ≤ 10%` |
| **F-07** | 必须超过目标金额 | 方向正确：目标为正则实际≥目标，目标为负则实际≤目标 |
| **F-08** | 目标金额可正可负 | 正值系统赢，负值系统输 |

### 3.2 波动体验需求 (MUST)

| ID | 需求描述 | 验收标准 |
|----|----------|----------|
| **V-01** | INTENSE 模式 - 剧烈波动 | 大奖倍率标准差 > 15，小奖倍率标准差 > 3 |
| **V-02** | GENTLE 模式 - 平稳波动 | 大奖倍率标准差 < 8，小奖倍率标准差 < 2 |
| **V-03** | 奖项分散性 | 连续中奖局数 ≤ 2，连续大奖局数 ≤ 1 |
| **V-04** | 随机性 | 不能连续3局以上中同类型奖（同为大奖或同为小奖） |

### 3.3 测试场景需求 (MUST)

| ID | 场景描述 | 输入 | 预期 |
|----|----------|------|------|
| **T-01** | 恒定投注-系统输 | 30局, 目标-5000, 投注=4 | 系统输≥4500, ≤5500 |
| **T-02** | 恒定投注-系统赢 | 30局, 目标+5000, 投注=4 | 系统赢≥4500, ≤5500 |
| **T-03** | 递增投注-系统输 | 30局, 目标-5000, 投注 4→500 | 系统输≥4500, ≤5500 |
| **T-04** | 递减投注-系统输 | 30局, 目标-5000, 投注 500→4 | 系统输≥4500, ≤5500 |
| **T-05** | 极端目标-系统大赢 | 30局, 目标+10000, 投注=100 | 系统赢≥9000, ≤11000 |
| **T-06** | 极端目标-系统大输 | 30局, 目标-10000, 投注=100 | 系统输≥9000, ≤11000 |

## 5. 验收判定矩阵

| 检查项 | 判定条件 | 权重 |
|--------|----------|------|
| 中奖率 ≤ 40% | `实际中奖率 ≤ 0.40` | 必选 |
| 大奖占比 = 50% | `大奖次数 / 中奖次数 = 0.50 (±10%)` | 必选 |
| 目标方向正确 | `符号一致或为0` | 必选 |
| 目标偏差 ≤ 10% | `|实际-目标|/|目标| ≤ 0.10` | 必选 |
| 局数内完成 | `gamesPlayed ≤ targetGames` | 必选 |
| 波动性符合模式 | V-01 或 V-02 满足 | 必选 |
| 分散性合规 | V-03, V-04 满足 | 必选 |

**通过条件**：所有必选项为 `true`

## 7. 附录

### 7.1 大奖/小奖定义

- **大奖 (Big Win)**: 倍率 **> 20**
- **小奖 (Small Win)**: 0 < 倍率 **≤ 20**
- **不中**: 倍率 = 0

### 7.2 VibeType 规格

| 模式 | 描述 | 大奖倍率分布 | 小奖倍率分布 |
|------|------|-------------|-------------|
| INTENSE | 剧烈波动 | 标准差 > 15 | 标准差 > 3 |
| GENTLE | 平稳波动 | 标准差 < 8 | 标准差 < 2 |

### 7.3 分散性规则

1. 连续中奖局数 ≤ 2
2. 连续大奖局数 ≤ 1
3. 不能连续3局以上中同类型奖
