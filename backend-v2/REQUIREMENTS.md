# 按局数完成目标系统输赢金额算法 - 需求文档

## 1. 概述

### 1.1 文档目的
本文档定义按局数完成目标系统输赢金额算法的功能需求、验收标准及测试用例。

### 1.2 核心目标
在指定局数内，通过控制中奖率和奖项分布，使系统输赢金额达到目标值（可正可负），同时提供不同的玩家波动体验。

### 1.3 关键术语定义

| 术语 | 定义 |
|------|------|
| 系统输赢金额 | `总投注 - 总派奖`，正数表示系统赢，负数表示玩家赢 |
| 目标金额 | 期望达成的系统输赢金额 |
| 大奖 | 倍率 ≥ 20 的奖项 |
| 小奖 | 倍率 < 20 的奖项 |
| 中奖率 | 中奖局数 / 总局数 |
| 波动体验 | 玩家感知到的输赢剧烈程度 |

## 2. 功能需求

### 2.1 接口约束（REQ-01, REQ-02）

**构造函数签名（不得改变）：**
```java
public Controller(int targetGames, BigDecimal targetSystemProfit,
                  Map<Double, Integer> rateWeightMap, VibeType vibeType,
                  List<BigDecimal> betList,
                  int gamesPlayed, int winCount, BigDecimal totalBet, BigDecimal totalPayout)
```

**calculateMultiplier方法签名（不得改变）：**
```java
public Double calculateMultiplier(BigDecimal playerBet)
```

### 2.2 目标达成约束（REQ-03, REQ-04）

| 指标 | 要求 |
|------|------|
| 完成偏差 | 实际系统输赢金额与目标金额的偏差不超过 ±10% |
| 完成方向 | 实际系统输赢金额必须"超过"目标（符号一致，数值绝对值≥目标） |
| 计算公式 | 系统输赢金额 = 总投注 - 总派奖 |

**验收条件：**
- `|实际金额 - 目标金额| / |目标金额| ≤ 10%`
- `实际金额 * 目标金额 > 0`（同向）

### 2.3 中奖率约束（REQ-05, REQ-06, REQ-07）

| 指标 | 要求 |
|------|------|
| 总中奖率上限 | ≤ 40% |
| 大奖(≥20x)占比 | 占总中奖局数的 20%（误差±5%） |
| 小奖(<20x)占比 | 约占总中奖局数的 80% |
| 分布均匀性 | 中奖事件应均匀分布在局数内 |

**计算公式：**
- `大奖局数 = round(总中奖局数 × 0.20)`
- `小奖局数 = 总中奖局数 - 大奖局数`
- `总中奖局数 = round(总局数 × 中奖率)`，中奖率 ∈ [0.20, 0.40]

### 2.5 波动体验（REQ-11, REQ-12, REQ-13）

#### 2.5.1 剧烈波动模式 (INTENSE)
| 特征 | 描述 |
|------|------|
| 大奖分布 | 大奖集中在少数局，中大奖时玩家赢很多 |
| 小奖分布 | 小奖也相对集中，但与大奖错开 |
| 玩家体感 | 剧烈起伏，大赢大输 |
| 实现方式 | 中奖时选择倍率偏向极端值（很大或很小） |

#### 2.5.2 平稳波动模式 (GENTLE)
| 特征 | 描述 |
|------|------|
| 大奖分布 | 大奖分散，小额大奖穿插其中 |
| 小奖分布 | 小奖均匀分布，倍率差异小 |
| 玩家体感 | 相对平稳，波动小 |
| 实现方式 | 中奖时选择倍率偏向中间值 |

#### 2.5.3 奖项分散性要求
- 避免连续多局中奖
- 避免连续多局中同类型奖
- 大奖和小奖应交错出现
- 使用随机打乱算法确保分散性

## 3. 非功能需求

### 3.1 性能需求
- 单局计算时间 < 10ms
- 支持最大10000局场景

### 3.2 随机性需求
- 使用种子随机避免可预测性
- 奖项选择需有随机因素

### 3.3 可恢复性
- 支持 snapshot/restore 机制
- 状态保存需包含所有关键变量
