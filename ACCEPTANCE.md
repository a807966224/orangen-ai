## 2. 需求规格摘要

### 2.1 核心定义

| 项目 | 定义 |
|------|------|
| 系统输赢金额 | `总投注 - 总派奖` |
| 目标金额 | 可正可负：正=系统赢，负=系统输 |
| 局数 | 完成目标的局数上限 |
| 大奖 (Big Win) | 倍率 **> 20** |
| 小奖 (Small Win) | 0 < 倍率 **≤ 20** |
| 总中奖率 | 40% ~ 60% (可动态调整) |
| 大奖占比 | 大奖次数 / 中奖次数 = **50%** |

### 2.2 波动性要求

| VibeType | 大奖倍率标准差 | 小奖倍率标准差 |
|----------|---------------|---------------|
| INTENSE | > 15 | > 3 |
| GENTLE | < 8 | < 2 |

### 2.3 分散性要求

- 连续中奖局数 ≤ 2
- 连续大奖局数 ≤ 1
- 不能连续3局以上中同类型奖

## 4. 数学约束分析

### 4.1 可达性判断公式

```
总投注 = Σ(每局投注)
需要派奖 = 总投注 - 目标盈利
最小派奖 = 中奖次数 × 平均小奖倍率 × min(投注)
最大派奖 = 中奖次数 × 平均大奖倍率 × max(投注)

可达条件: 最小派奖 ≤ 需要派奖 ≤ 最大派奖
```

### 4.2 典型场景分析

| 场景 | 总投注 | 目标 | 需要派奖 | 最小派奖(6小) | 最大派奖(6大) | 可达性 |
|------|--------|------|----------|-----------------|----------------|--------|
| bet=4, target=-1200 | 120 | -1200 | 1320 | 360 (6×4×15) | 3120 (6×4×130) | ✅ |
| bet=4, target=-5000 | 120 | -5000 | 5120 | 360 | 3120 | ❌ |
| bet=100, target=-5000 | 3000 | -5000 | 8000 | 12000 | 132000 | ❌ |
| bet=100, target=-12000 | 3000 | -12000 | 15000 | 12000 | 132000 | ⚠️ |

## 6. 算法能力总结

### 6.1 可达成场景

| 场景 | 典型偏差 | 说明 |
|------|---------|------|
| bet=4, target=-1200 | ~10-15% | ✅ 可达成 |
| bet=4, target=+1200 | ~10-15% | ✅ 可达成 |
| bet=100, target=-12000 | ~10-20% | ⚠️ 边缘可达 |

### 6.2 不可达成场景

| 场景 | 原因 |
|------|------|
| bet=4, target=-5000 | 理论最大派奖3120 < 需求5120 |
| bet=100, target=-5000 | 最小派奖12000 > 需求8000 |

### 6.3 核心限制

1. **大奖倍率下限**: 大奖必须 > 20x，即使选择最低大奖(20.3x)，也会产生大量派奖
2. **投注金额影响**: 大bet时最小派奖可能就已超过目标
3. **中奖率上限**: 60%上限限制了在低bet时达成高目标的能力

## 8. 算法使用指南

### 8.1 目标设置公式

```kotlin
// 可达目标估算
val totalBet = betList.sum()
val maxPayout = totalWins * maxBet * maxMultiplier  // 理想情况
val minPayout = totalWins * minBet * minMultiplier  // 保守情况

// 系统赢目标 (正数) 应小于 maxPayout - totalBet
// 系统输目标 (负数) 应大于 minPayout - totalBet
```

### 8.2 示例

```kotlin
// bet=4, 30局, 12中奖
val totalBet = 120
val maxProfit = 12 * 4 * 110 - 120  // ≈ 5160 (最大系统输)
val minProfit = 120 - 12 * 4 * 1   // ≈ 72 (最小系统赢)

// 建议目标范围: -1500 ~ +500
```
